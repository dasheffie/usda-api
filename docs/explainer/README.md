# Explainer video — source

45-second HyperFrames composition that renders `usda-api-explainer.mp4` — the video embedded at the top of the repo's main [README](../../README.md).

## Regenerate from scratch

```bash
cd docs/explainer
mkdir -p audio
npx hyperframes tts narration.txt --voice af_nova --output audio/narration.wav
npx hyperframes render --quality standard --output renders/usda-api-explainer.mp4
```

Requires Node ≥ 22 and FFmpeg. First `tts` run downloads the Kokoro-82M model (~86 MB) and caches it.

## Layout

- `index.html` — root composition, 6 scenes, all inline (no sub-compositions)
- `DESIGN.md` — visual identity (Data Drift: dark canvas, cyan + electric purple + amber, JetBrains Mono)
- `narration.txt` — TTS source script (synthesized to `audio/narration.wav` at render time)
- `hyperframes.json` — project config

The hero chart in scene 5 is referenced from the repo root at `../../aggregate-trajectories.png` rather than duplicated here.

## Build artifacts (gitignored)

- `audio/narration.wav` — generated by `tts`
- `transcript.json` — word-level timings (generated by `transcribe`, not required by the current composition which uses phrase-level captions baked into `index.html`)
- `renders/*.mp4` — rendered output
- `.hyperframes/`, `meta.json`, `node_modules/` — tool state
